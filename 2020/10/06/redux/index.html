参考链接 — 关于redux的一份食用说明😋

redux/react-redux

redux: js的状态管理

react-redux: 在react中更容易使用redux — connect,provide

redux

  • 存放数据 — store = createStore(reducer)
  • 定义如何更新数据 — reducer(oldState, action) => state更新
  • 外界可以访问数据 — store.getState()
  • 外界也能修改数据 — dispatch(action) => state更新
  • 订阅state的更新，取消订阅 — subscribe(){ return unsubscribe() }

目录结构相关

🎉看懂食用说明了​​，我们需要补充目录结构。

  • reducers文件夹：用于存放所有reducer文件，在默认的rootReducer—index.js中组合其他reducer

  • components文件夹

  • actions文件夹：因为我们不可能在根组件中规定所有的事件触发的action操作，把actions单独拎出来可以使目录结构更清楚。

  • constants文件夹：存放actions常量

  • 借助<Provider>实现store全局可用（<Provider>底层是通过上下文context实现的）。在每个组件中，将state、dispatch映射进props。（<Provider>、connect、mapStateToProps、mapDispatchToProps）
